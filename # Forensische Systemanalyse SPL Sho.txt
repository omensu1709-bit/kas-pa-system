## 1. Systemübersicht

Das SPL Short Tool ist ein hochspezialisiertes Kryptowährungs-Handelssystem, das sich auf die Vorhersage von Kursrückgängen bei SPL-Token (Solana Protocol Tokens) konzentriert. Das System basiert auf dem Prinzip der **Informationshoheit** (#INFO) – anstatt gegen HFT-Systeme (High-Frequency Trading) anzutreten, nutzt das System das Wissen über das Verhalten von HFC-Bots (High-Frequency Crypto Bots), um strukturelle Marktbewegungen zu prognostizieren.

### 1.1 Kernprinzipien

- **Ziel:** Kursbewegungen größer als 3% nach unten
- **Fokus:** Die „sicheren 20 Prozent" – nicht die ersten 5% einer Bewegung, sondern die nachfolgende strukturelle Korrektur
- **Asset-Pool:** 81 SPL Coins werden kontinuierlich überwacht
- **Turnus:** Alle 60 Minuten werden 20 Coins mit dem höchsten Short-Potential ausgewählt
- **Datenquellen:** ChainStack Yellowstone gRPC, Binance, Bybit
- **Hardware:** Hetzner Ryzen 7 3700X, 64GB ECC RAM, 1 Gbit Netzwerk

## 4. Identifizierte Probleme

### 4.1 Kritisch

| Problem | Auswirkung | Lösungsvorschlag |
|---------|------------|------------------|
| Trigger-Score Sättigung | System kann nicht selektieren | Guillotine-Threshold anheben |
| Ghost-Pump-Dominanz | Falsche Positive | Automatisches Asset-Muting |
| Leere Paper-Trade-Tabelle | Keine Performance-Validierung | Evaluator-Prozess prüfen |

### 4.2 Mittlere Priorität

- **Datenqualität:** Timestamp-Entropie-Checks zeigen vereinzelt synthetische Daten
- **Modell-Drift:** Feature-Drift-Monitoring nicht aktiv
- **Redis-Verbindung:** Socket-Verbindung fehlgeschlagen, TCP-Fallback erforderlich

## 5. Empfohlene nächste Schritte

### 5.1 Sofortmaßnahmen (≤ 1 Stunde)

1. **Guillotine-Calibration:**
   - Threshold anheben: `trigger_score > 0.8` statt bisheriger Werte
   - Kombination mit `conviction_score > 0.15`

2. **Asset-Muting aktivieren:**
   ```bash
   redis-cli SADD trinity:command:muted_symbols ENAUSDT DOGEUSDT PIPPINUSDT SUIUSDT ONDOUSDT
   redis-cli EXPIRE trinity:command:muted_symbols 3600
   ```

3. **Ghost-Prozess bereinigen:**
   ```bash
   ps aux | grep kas_evaluator
   sudo kill -9 <PID>
   ```

### 5.2 Kurzfristige Maßnahmen (≤ 7 Tage)

- Wöchentliches Modell-Retraining mit Conformal Prediction
- Automatisierte Feature-Drift-Überwachung via SHAP-Baseline
- Cross-Asset-Korrelationsmodul aktivieren
